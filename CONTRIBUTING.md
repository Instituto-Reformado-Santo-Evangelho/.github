# 🤝 Contribuindo para o Instituto Reformado Santo Evangelho

> *"Assim, pois, cada um de nós dará contas de si mesmo a Deus."* - **Romanos 14:12**

Agradecemos seu interesse em contribuir para os projetos do **Instituto Reformado Santo Evangelho**! Este guia ajudará você a participar de nossa missão de usar a tecnologia para servir ao Reino de Deus.

## 🙏 **Princípios de Contribuição Cristã**

Antes de contribuir, pedimos que você considere estes princípios baseados nas Escrituras:

### **❤️ Amor Fraternal**
*"Nisto conhecerão todos que sois meus discípulos: se tiverdes amor uns aos outros."* - **João 13:35**
- Trate todos os colaboradores com amor e respeito
- Seja paciente com diferentes níveis de experiência
- Ofereça ajuda construtiva em revisões de código

### **🕊️ Mansidão e Humildade**
*"Nada façais por partidarismo ou vanglória, mas por humildade, considerando cada um os outros superiores a si mesmo."* - **Filipenses 2:3**
- Aceite feedback de forma humilde
- Reconheça que erros fazem parte do aprendizado
- Valorize as contribuições de outros, independente do tamanho

### **⚖️ Verdade e Integridade**
*"Comprai a verdade e não a vendais; também a sabedoria, a disciplina e o entendimento."* - **Provérbios 23:23**
- Seja honesto sobre seu nível de conhecimento
- Documente mudanças claramente
- Teste seu código antes de submeter

### **🏗️ Edificação Mútua**
*"Portanto, consolai-vos uns aos outros e edificai-vos reciprocamente, como também estais fazendo."* - **1 Tessalonicenses 5:11**
- Foque em soluções que beneficiem toda a comunidade
- Compartilhe conhecimento e recursos
- Celebre as conquistas de outros colaboradores

## 🚀 **Processo de Contribuição**

### **1. 🍴 Fork e Clone**
```bash
# Fork o repositório no GitHub
# Clone seu fork
git clone https://github.com/SEU-USUARIO/NOME-DO-REPO.git
cd NOME-DO-REPO

# Adicione o repositório original como upstream
git remote add upstream https://github.com/Instituto-Reformado-Santo-Evangelho/NOME-DO-REPO.git
```

### **2. 🌿 Crie uma Branch**
```bash
# Crie uma branch descritiva
git checkout -b feature/nova-funcionalidade
# ou
git checkout -b fix/correcao-bug
# ou  
git checkout -b theology/revisao-mateus-5
```

### **3. ✍️ Faça suas Mudanças**
- Siga os padrões de código do projeto
- Escreva mensagens de commit claras
- Adicione testes quando apropriado
- Documente mudanças significativas

### **4. 🧪 Teste Localmente**
```bash
# Para repositórios com testes automatizados
./run-tests.sh

# Para módulos TheWord
# Teste no software TheWord antes de submeter
```

### **5. 📤 Submeta um Pull Request**
- Use o template de PR quando disponível
- Descreva claramente as mudanças
- Referencie issues relacionadas
- Seja paciente durante a revisão

## 📋 **Padrões de Commit**

Use mensagens de commit claras e descritivas:

```bash
# Bom ✅
git commit -m "✨ Adiciona verificação de encoding UTF-8 nos módulos"
git commit -m "🐛 Corrige problema de formatação em Mateus 5:3"
git commit -m "📝 Melhora documentação de instalação do TheWord"

# Evite ❌
git commit -m "fix"
git commit -m "update"
git commit -m "mudanças"
```

### **🏷️ Prefixos Sugeridos**
- `✨` - Nova funcionalidade
- `🐛` - Correção de bug
- `📝` - Documentação
- `🎨` - Melhoria de código/estrutura
- `⚡` - Melhoria de performance
- `🧪` - Testes
- `📖` - Conteúdo bíblico/teológico

## 🔍 **Processo de Revisão**

### **📖 Revisão Teológica**
Para mudanças em conteúdo bíblico:
1. Revisão técnica inicial
2. Revisão teológica por pastor/acadêmico
3. Verificação final antes do merge

### **💻 Revisão Técnica**
Para mudanças de código:
1. Revisão automática (GitHub Actions)
2. Revisão por desenvolvedor experiente
3. Testes manuais quando necessário
